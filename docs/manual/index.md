# Manual de Git e GitHub

Bem-vindo ao manual de uso de Git e GitHub.

Este guia foi criado para padronizar o fluxo de trabalho da equipe, garantindo:

- Segurança no desenvolvimento
- Organização das contribuições
- Rastreabilidade das mudanças
- Facilidade para novos membros

---

## 📌 O que você encontrará aqui

- Como usar branches corretamente
- Como fazer commits padronizados
- Como abrir Pull Requests
- Fluxo oficial do projeto
- Boas práticas obrigatórias

---

## ⚠️ Importante

Este projeto segue regras específicas de versionamento. Não seguir o padrão pode causar:

- Conflitos de código
- Perda de trabalho
- Quebra do projeto

---

## Comece por aqui

Se você é iniciante, siga essa ordem (ela reflete nosso fluxo de trabalho real):

1. [Objetivo](index.md) — Visão geral do propósito do guia e do que se espera garantir com o processo definido.
2. [Estratégia de Branches](branches.md) — Explica como as branches são organizadas no projeto e o padrão correto de nomenclatura.
3. [Commits](commits.md) — Define o padrão obrigatório de mensagens e os tipos de commit utilizados no projeto.
4. [Envio de Alterações](envio-alteracoes.md) — Guia prático com o passo a passo para criar branch, commitar e enviar alterações ao repositório.
5. [Pull Request (PR)](pull-request.md) — Orienta como abrir um PR corretamente e o que deve ser incluído na descrição.
6. [Fluxo de Pull Request](fluxo-pr.md) — Mostra o caminho que o código percorre desde a criação do PR até a integração na `main`.
7. [Revisão e Merge](revisao-merge.md) — Explica como funciona a revisão de código e os critérios necessários para aprovação e merge.
8. [Comandos Git](comandos-git.md) — Reúne os principais comandos Git utilizados no dia a dia do projeto.
9. [Boas Práticas](boas-praticas.md) — Lista recomendações essenciais e erros que devem ser evitados durante o desenvolvimento.
10. [Resumo](resumo.md) — Apresenta o ciclo completo de uma tarefa, do início até a integração final no projeto.



---

## Princípio fundamental

Nunca trabalhar direto na branch `main`.
