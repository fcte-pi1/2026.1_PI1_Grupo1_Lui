# Boas Práticas

## ✔ Sempre fazer

- Criar uma branch para cada tarefa
- Fazer commits pequenos e frequentes
- Referenciar a issue em todo commit
- Atualizar a branch antes de começar a trabalhar
- Revisar o trabalho de outros membros

---

## ❌ Nunca fazer

- Commit direto na `main`
- Mensagens de commit vagas ou genéricas
- Trabalhar sem branch própria
- Fazer merge sem revisão
- Usar `--force` sem entender as consequências

---

## Dica

Commits pequenos e objetivos = histórico mais claro = muito menos dor de cabeça na hora de resolver conflitos.
